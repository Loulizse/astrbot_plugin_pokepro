# 更新日志

## v2.1.0

- 新增语音回复戳一戳能力。现在被戳后的随机响应模块中可以直接配置语音音频池，命中后会从池中随机选取一个音频文件发送，和反戳、LLM 回复、QQ 表情、表情包、禁言、触发命令等响应方式一样参与统一的权重抽取。
- 表情包配置新增 `meme.paths` 路径池支持。除了原有的 `meme.pool` 单文件列表外，现在还可以直接填写目录路径或文件路径，插件启动时会自动递归读取目录下的图片文件，并与原有图片池合并去重后统一作为可选表情包池使用；相对路径与绝对路径均可，适合直接复用图库插件或本地现成图片目录。
- 新增 LLM 工具 `llm_poke_user`。语言模型现在可以在合适的对话场景下主动调用该工具，对指定 QQ 用户发起戳一戳操作，并可传入戳的次数；工具会自动校验目标 QQ 号格式、阻止戳机器人自己，并将次数限制在插件配置的 `poke_max_times` 范围内，避免误调用导致过量发送。
- 补充插件元数据声明。在 `metadata.yaml` 中新增 `support_platforms` 与 `astrbot_version` 字段，明确该插件当前面向 `aiocqhttp` 平台适配器运行，并声明最低依赖版本为 `AstrBot v4.13.0`，便于插件市场、加载器和后续兼容性检查正确识别插件适用范围。
